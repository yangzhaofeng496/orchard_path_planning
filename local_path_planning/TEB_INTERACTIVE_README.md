# TEB 交互式演示说明

## 功能介绍

这是一个基于 TEB (Timed Elastic Band) 算法的交互式路径规划演示程序。你可以：

- **拖动起点和终点**：实时调整规划任务
- **拖动障碍物**：观察 TEB 如何动态避障
- **添加新障碍物**：测试算法在复杂环境中的表现
- **查看优化轨迹**：包含每个节点的方向箭头和速度标注

## 运行方法

```bash
cd /Users/yangzhaofeng/VsCodeProject/orchard_path_planning/local_path_planning
python teb_interactive.py
```

## 可视化说明

### 场景元素

- **绿色圆 + 箭头**：起点位置和初始朝向
- **红色圆 + 箭头**：终点位置和目标朝向
- **橙色圆**：障碍物（可拖动）
- **蓝色轨迹线**：TEB 优化后的路径
- **蓝色圆点**：TEB 节点位置
- **深蓝色箭头**：每个节点的运动方向
- **数字标注**：每个节点的速度（单位：m/s）
- **蓝色矩形**：车辆轮廓（部分节点）

### 信息面板

左上角显示：
- 规划状态（成功/失败及原因）
- 起点和终点坐标
- 障碍物数量
- TEB 节点数量
- 暂停状态

## TEB 算法特点

TEB 将局部规划转化为非线性优化问题：

### 优化目标
1. **时间最优**：最小化总运行时间
2. **障碍物避障**：保持安全距离
3. **运动学约束**：满足 Ackermann 车辆模型
4. **平滑性**：减少加速度和角速度变化
5. **路径跟踪**：靠近全局路径

### 优化方法
- 使用 **SciPy SLSQP** (Sequential Least Squares Programming)
- 优化变量：`[x₀, y₀, θ₀, Δt₀, x₁, y₁, θ₁, Δt₁, ..., xₙ, yₙ, θₙ, Δtₙ]`
- 每 0.5 秒重新规划一次

## 配置文件

TEB 参数配置在：
```
configs/teb_config.yaml
```

主要参数：
- `num_samples`: TEB 节点数量（默认 10）
- `max_iterations`: 最大优化迭代次数（默认 100）
- `w_obstacle`: 障碍物避障权重（默认 5.0）
- `w_kinematics`: 运动学约束权重（默认 3.0）
- `obstacle_min_dist`: 最小安全距离（默认 0.5m）

## 常见问题

### Q: 为什么有时候规划失败？
A: 可能原因：
- 障碍物完全堵住了路径
- 起点或终点太靠近边界
- 优化器未能在迭代次数内收敛

### Q: 速度标注是什么意思？
A: 显示的数字是该节点的瞬时速度（m/s），由相邻节点间距离和时间间隔计算得出。

### Q: 如何调整优化性能？
A: 修改 `configs/teb_config.yaml`：
- 增加 `max_iterations` 提高收敛机会
- 减少 `num_samples` 加快计算速度
- 调整权重 `w_*` 改变优化侧重点

## 与 DWA 的对比

| 特性 | TEB | DWA |
|------|-----|-----|
| 方法 | 连续优化 | 采样搜索 |
| 时间考虑 | 时间最优 | 固定时间窗口 |
| 轨迹平滑度 | 高 | 中等 |
| 计算复杂度 | 较高 | 较低 |
| 局部最优 | 可能陷入 | 贪心选择 |
