## 1. 系统概述

### 1.1 整体架构

本系统采用**分层规划架构**：

```
┌─────────────────────────────────────────────────────────┐
│  全局规划层: OAG-HRRT* (Orchard Adaptive Guidance)     │
│  - 目标导向采样 + 自适应矩形引导                        │
│  - 障碍物聚类与切线逃逸                                 │
│  - 路径优化器 (Path Shortcut + Curvature Smoothing)    │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  局部规划层: 三种可切换的局部规划器                     │
│  ├─ DWA (Dynamic Window Approach)                       │
│  ├─ TEB (Timed Elastic Band)                           │
│  └─ Adaptive TEB-DWA (自适应融合)                       │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  执行层: Ackermann 车辆运动学模型                        │
│  - 速度控制 + 转向控制                                   │
│  - 实时避障 + 轨迹跟踪                                   │
└─────────────────────────────────────────────────────────┘
```

### 1.2 核心特点

- **统一接口**: 三种局部规划器共享相同的接口 (`LocalPlanner`)
- **模块化设计**: 全局规划与局部规划完全解耦
- **配置驱动**: 所有参数通过 YAML 配置文件管理
- **实时性**: 控制周期 0.05-0.10 秒，满足实时要求

## 3. 三种局部规划器详解

### 3.1 DWA (Dynamic Window Approach)

#### 3.1.1 算法原理

DWA 是一种**基于采样的快速局部规划方法**，通过在动态窗口内采样控制指令并评估轨迹代价来选择最优控制。

**核心思想**:
1. 根据车辆加速度限制计算可达速度范围 (动态窗口)
2. 在该窗口内采样多组 (速度, 转向角) 组合
3. 使用 Ackermann 运动学模型预测每组控制的轨迹
4. 计算每条轨迹的代价 (目标距离 + 路径跟踪 + 障碍物 + 平滑性)
5. 选择代价最小且无碰撞的轨迹

#### 3.1.2 算法流程

```python
def dwa_plan(state, obstacles, global_path):
    # 1. 计算动态窗口
    v_min = max(0, state.speed - max_decel * dt)
    v_max = min(max_speed, state.speed + max_accel * dt)
    δ_min = max(-max_steer, state.steering - max_steer_rate * dt)
    δ_max = min(max_steer, state.steering + max_steer_rate * dt)
    
    # 2. 采样控制空间
    for v in linspace(v_min, v_max, speed_samples):
        for δ in linspace(δ_min, δ_max, steer_samples):
            # 3. 预测轨迹
            trajectory = predict_ackermann_trajectory(v, δ, predict_time)
            
            # 4. 碰撞检测
            if has_collision(trajectory, obstacles):
                continue
            
            # 5. 计算代价
            cost = (
                w_goal * goal_distance(trajectory) +
                w_path * path_tracking_error(trajectory, global_path) +
                w_heading * heading_error(trajectory) +
                w_obstacle * obstacle_clearance(trajectory, obstacles) +
                w_speed * speed_penalty(v) +
                w_steering * steering_penalty(δ) +
                w_progress * progress_reward(trajectory)
            )
            
            # 6. 更新最优解
            if cost < best_cost:
                best_trajectory = trajectory
                best_control = (v, δ)
    
    return best_control, best_trajectory
```

#### 3.1.3 代价函数

DWA 使用**多目标加权代价函数**，默认权重如下：

| 代价项 | 权重 | 含义 |
|--------|------|------|
| `goal_cost` | 3.0 | 与局部目标点的距离 |
| `path_cost` | 2.0 | 偏离全局路径的距离 |
| `heading_cost` | 0.8 | 航向误差 |
| `obstacle_cost` | 3.5 | 障碍物接近惩罚 |
| `speed_cost` | 0.45 | 速度偏好 (鼓励接近最大速度) |
| `steering_cost` | 0.2 | 转向角惩罚 (鼓励直行) |
| `steering_change_cost` | 0.5 | 转向变化惩罚 (平滑性) |
| `progress_reward` | 0.04 | 路径进度奖励 |

#### 3.1.4 配置参数

```yaml
dwa:
  dt: 0.05                # 控制周期 (秒)
  predict_time: 2.8       # 预测时域 (秒)
  speed_samples: 7        # 速度采样数
  steer_samples: 19       # 转向采样数
  max_speed: 2.0          # 最大速度 (m/s)
  max_accel: 1.0          # 最大加速度 (m/s²)
  max_decel: 1.0          # 最大减速度 (m/s²)
  max_steer_rate_deg: 70.0  # 最大转向角速度 (度/秒)
```

#### 3.1.5 优缺点

**优点**:
- ✅ **计算快速**: 7×19=133 条轨迹，适合实时应用
- ✅ **动态避障**: 天然处理动态障碍物
- ✅ **易于调试**: 代价权重直观，可视化友好
- ✅ **鲁棒性强**: 即使全局路径质量差也能工作

**缺点**:
- ❌ **轨迹不平滑**: 离散采样导致控制不连续
- ❌ **局部最优**: 可能陷入局部极小值
- ❌ **前瞻有限**: 预测时域受限 (通常 1-3 秒)
- ❌ **参数敏感**: 权重需要针对场景精细调优

### 3.2 TEB (Timed Elastic Band)

#### 3.2.1 算法原理

TEB 是一种**基于优化的局部规划方法**，将路径规划转化为**非线性约束优化问题**。

**核心思想**:
1. 将路径表示为一系列时空节点 `(x, y, θ, Δt)`
2. 构建包含多个目标的代价函数
3. 使用非线性优化器 (SLSQP) 同时优化路径形状和时间分配
4. 满足车辆运动学约束 (Ackermann 模型)

**关键区别于 DWA**: TEB 优化整条轨迹，而 DWA 只选择最优控制。

#### 3.2.2 算法流程

```python
def teb_plan(state, obstacles, global_path):
    # 1. 初始化/更新 TEB 节点
    if not teb_nodes:
        teb_nodes = initialize_teb_from_path(state, global_path, num_samples=20)
    else:
        teb_nodes = update_teb_sliding_window(state, global_path)
    
    # 2. 构建优化变量
    x0 = [node.x, node.y, node.yaw, node.dt for node in teb_nodes]
    
    # 3. 设置边界约束
    bounds = []
    for i, node in enumerate(teb_nodes):
        if i == 0:
            # 第一个节点固定在当前位置
            bounds += [(node.x, node.x), (node.y, node.y), (node.yaw, node.yaw)]
        else:
            # 其他节点可在空间内移动
            bounds += [(x_min, x_max), (y_min, y_max), (-π, π)]
        bounds += [(0.01, 2.0)]  # 时间间隔边界
    
    # 4. 执行非线性优化
    result = scipy.optimize.minimize(
        fun=objective_function,
        x0=x0,
        method='SLSQP',
        bounds=bounds,
        options={'maxiter': max_iterations}
    )
    
    # 5. 解包优化结果
    optimized_teb_nodes = unpack_variables(result.x)
    
    # 6. 硬碰撞复核
    if has_vehicle_collision(optimized_teb_nodes, obstacles):
        return None  # 拒绝不安全轨迹
    
    # 7. 提取控制指令
    control = extract_control_from_teb(optimized_teb_nodes)
    
    return control, optimized_teb_nodes
```

#### 3.2.3 目标函数

TEB 的目标函数是**多项代价的加权和**：

```python
J_total = w_time * J_time +                    # 时间最优
          w_obstacle * J_obstacle +            # 障碍物避障
          w_kinematics * J_kinematics +        # 运动学约束
          w_acceleration * J_acceleration +    # 加速度平滑
          w_omega * J_omega +                  # 角速度约束
          w_path * J_path                      # 路径跟踪
```

**各项代价详解**:

1. **时间代价** (`w_time = 1.0`)
   ```python
   J_time = Σ Δt_i  # 总时间最小化
   ```

2. **障碍物代价** (`w_obstacle = 50.0`)
   ```python
   for node in teb_nodes:
       for obs in obstacles:
           dist = vehicle_obstacle_clearance(node, obs)
           if dist < min_dist:
               J_obstacle += 1000 * (min_dist - dist)²
           elif dist < influence_dist:
               J_obstacle += 10 * (influence_dist - dist)
   ```

3. **运动学代价** (`w_kinematics = 20.0`)
   ```python
   # 确保路径符合 Ackermann 约束
   for i in range(len(nodes) - 1):
       v = distance(nodes[i], nodes[i+1]) / Δt_i
       if v > max_speed:
           J_kinematics += (v - max_speed)²
       
       required_steer = calculate_ackermann_steer(nodes[i], nodes[i+1])
       if required_steer > max_steer:
           J_kinematics += 100 * (required_steer - max_steer)²
   ```

4. **加速度代价** (`w_acceleration = 5.0`)
   ```python
   # 平滑性
   for i in range(len(nodes) - 2):
       v1 = speed(nodes[i], nodes[i+1])
       v2 = speed(nodes[i+1], nodes[i+2])
       acc = (v2 - v1) / ((Δt_i + Δt_{i+1}) / 2)
       J_acceleration += 0.1 * acc²
   ```

5. **角速度代价** (`w_omega = 10.0`)
   ```python
   for i in range(len(nodes) - 1):
       Δyaw = normalize_angle(nodes[i+1].yaw - nodes[i].yaw)
       omega = abs(Δyaw / Δt_i)
       if omega > max_omega:
           J_omega += (omega - max_omega)²
   ```

6. **路径跟踪代价** (`w_path = 1.0`)
   ```python
   for node in teb_nodes[1:]:  # 跳过起点
       min_dist = min(distance(node, ref) for ref in global_path)
       J_path += min_dist²
   ```

#### 3.2.5 优缺点

**优点**:
- ✅ **轨迹平滑**: 连续优化产生平滑轨迹
- ✅ **全局最优**: 在约束下寻找最优解
- ✅ **时间最优**: 同时优化路径和速度分布
- ✅ **理论基础**: 基于优化理论，可证明收敛性

**缺点**:
- ❌ **计算耗时**: 优化迭代需要 10-50 次，耗时 50-200ms
- ❌ **初值敏感**: 需要良好的初始轨迹
- ❌ **收敛不保证**: 复杂场景可能不收敛
- ❌ **实时性差**: 不适合高频控制 (> 10Hz)

### 3.3 Adaptive TEB-DWA (自适应融合)

#### 3.3.1 算法原理

Adaptive TEB-DWA 是**本项目的核心创新**，通过**迭代反馈机制**融合 TEB 和 DWA 的优势。

**设计思想**:
- **TEB 负责生成**: 产生平滑、时间最优的候选轨迹
- **DWA 负责评估**: 验证轨迹的运动学可达性和安全性
- **参数管理器**: 根据 DWA 反馈动态调整 TEB 权重
- **自适应窗口**: 根据环境密度和路径曲率动态调整规划范围

**核心循环**:
```
第1轮: TEB 生成 → DWA 评估 → 不可行
第2轮: 调整权重 (↑ 障碍物权重) → TEB 重新生成 → DWA 评估 → 不可行
第3轮: 继续调整 → TEB 生成 → DWA 评估 → 可行 ✓
```

#### 3.3.2 算法流程

```python
def adaptive_teb_dwa_plan(state, obstacles, global_path):
    best_candidate = None
    adjustments = []
    
    # 最多迭代 N 轮 (默认 6 轮)
    for iteration in range(1, max_feedback_iterations + 1):
        # 1. 自适应窗口选择
        window = adaptive_window_selector.select(
            global_path, state, obstacles, current_window_scale
        )
        # window 包含: lookahead_distance, obstacle_density, path_curvature
        
        # 2. TEB 优化
        teb_result = teb_interface.optimize(
            window.reference_path, state, obstacles
        )
        
        # 3. DWA 评估
        if teb_result is None:
            evaluation = DWAEvaluation(
                score=0.0, feasible=False, reason="TEB_FAILED"
            )
        else:
            evaluation = dwa_evaluator.evaluate(
                teb_result, state, obstacles
            )
        
        # 4. 记录最佳候选
        if evaluation.score > best_score:
            best_candidate = (evaluation, teb_result, window)
        
        # 5. 如果可行，立即返回
        if evaluation.feasible:
            return success_result(teb_result, evaluation, iteration)
        
        # 6. 参数调整（为下一轮做准备）
        if iteration < max_feedback_iterations:
            adjustment = parameter_manager.adjust(
                evaluation, window.obstacle_density
            )
            adjustments.append(adjustment)
            
            # 更新 TEB 权重
            teb_interface.update_weights(
                adjustment.obstacle_weight,
                adjustment.kinematic_weight,
                adjustment.omega_weight
            )
    
    # 7. 所有轮次都失败，返回最佳候选（如果允许）
    if accept_best_score and best_candidate:
        return partial_success_result(best_candidate)
    
    return failure_result()
```

#### 3.3.3 四大核心组件

##### (1) AdaptiveWindowSelector - 自适应窗口选择器

根据**环境密度、路径曲率、自由空间、速度**动态调整前瞻距离：

```python
def select_window(global_path, state, obstacles, scale):
    # 基础前瞻距离
    base_lookahead = 7.0
    
    # 1. 计算障碍物密度
    sensing_radius = 6.0
    nearby_obstacles = count_obstacles_within(state, obstacles, sensing_radius)
    density = nearby_obstacles / (π * sensing_radius²)
    
    # 2. 计算路径曲率
    curvature = compute_path_curvature(global_path, state, lookahead=5.0)
    
    # 3. 计算自由空间
    free_space = min_obstacle_distance(state, obstacles)
    
    # 4. 综合调整
    lookahead = base_lookahead
    lookahead += speed_gain * state.speed         # 速度越快，前瞻越远
    lookahead -= density_gain * density           # 密度越大，前瞻越近
    lookahead -= curvature_gain * curvature       # 曲率越大，前瞻越近
    lookahead += free_space_gain * free_space     # 自由空间越大，前瞻越远
    
    # 5. 窗口缩放
    lookahead *= scale
    
    # 6. 边界限制
    lookahead = clip(lookahead, min_lookahead=3.0, max_lookahead=12.0)
    
    return WindowSelection(
        lookahead_distance=lookahead,
        reference_path=extract_path_segment(global_path, state, lookahead),
        obstacle_density=density,
        path_curvature=curvature
    )
```

**配置参数**:
```yaml

  max_lookahead: 12.0          # 最大前瞻 (米)
  base_lookahead: 7.0          # 基础前瞻 (米)
  sensing_radius: 6.0          # 感知半径 (米)
  density_reference: 0.22      # 参考密度 (个/m²)
  curvature_reference: 0.25    # 参考曲率 (1/m)
  speed_gain: 1.2              # 速度增益
  density_gain: 3.0            # 密度惩罚
  curvature_gain: 3.0          # 曲率惩罚
  free_space_gain: 1.5         # 自由空间增益
```

##### (2) DWAFeedbackEvaluator - DWA 反馈评估器

使用 DWA 评估 TEB 生成的轨迹：

```python
def evaluate(teb_trajectory, state, obstacles):
    # 将 TEB 轨迹转换为 DWA 可识别的全局路径
    dwa_reference_path = teb_trajectory
    
    # 使用 DWA 规划
    dwa_result = dwa_planner.plan(state, obstacles)
    
    if dwa_result.best is None:
        return DWAEvaluation(
            score=0.0,
            feasible=False,
            reason="DWA_NO_SOLUTION"
        )
    
    # 评估 TEB 轨迹与 DWA 最优轨迹的差异
    tracking_error = compute_tracking_error(
        teb_trajectory, dwa_result.best.trajectory
    )
    kinematic_violation = check_kinematic_constraints(
        teb_trajectory, state
    )
    obstacle_risk = min_obstacle_clearance(teb_trajectory, obstacles)
    
    # 计算综合得分
    score = 1.0 / (1.0 + tracking_error + kinematic_violation)
    
    # 判断可行性
    feasible = (
        tracking_error < threshold and
        kinematic_violation < threshold and
        obstacle_risk > min_safe_distance
    )
    
    return DWAEvaluation(
        score=score,
        tracking_error=tracking_error,
        kinematic_violation=kinematic_violation,
        obstacle_risk=obstacle_risk,
        feasible=feasible,
        reason="OK" if feasible else "TRACKING_ERROR"
    )
```

##### (3) ParameterManager - 参数管理器

根据 DWA 反馈**有界调整** TEB 权重：

```python
def adjust_parameters(evaluation, obstacle_density):
    # 1. 分析失败原因
    if evaluation.tracking_error > threshold:
        # 跟踪误差大 → 增加路径跟踪权重，降低其他权重
        kinematic_multiplier = 1.35
        obstacle_multiplier = 1.0
    elif evaluation.kinematic_violation > threshold:
        # 运动学违规 → 增加运动学权重
        kinematic_multiplier = 1.5
        obstacle_multiplier = 1.0
    elif evaluation.obstacle_risk < min_safe_distance:
        # 障碍物太近 → 大幅增加障碍物权重
        obstacle_multiplier = 1.5
        kinematic_multiplier = 1.0
    else:
        # 其他情况 → 轻微调整
        obstacle_multiplier = 1.2
        kinematic_multiplier = 1.1
    
    # 2. 应用乘数（有界）
    new_obstacle_weight = clip(
        current_obstacle_weight * obstacle_multiplier,
        min=10.0, max=250.0
    )
    new_kinematic_weight = clip(
        current_kinematic_weight * kinematic_multiplier,
        min=5.0, max=100.0
    )
    
    # 3. 窗口缩放调整
    if obstacle_density > density_reference:
        window_scale = max(0.55, window_scale * shrink_factor)
    else:
        window_scale = min(1.35, window_scale * expand_factor)
    
    return ParameterAdjustment(
        obstacle_weight=new_obstacle_weight,
        kinematic_weight=new_kinematic_weight,
        window_scale=window_scale
    )
```

##### (4) TEBInterface - TEB 接口封装

封装 TEB 规划器，提供动态权重更新：

```python
class TEBInterface:
    def __init__(self, teb_config, bounds):
        self.config = teb_config
        self.last_planner = None
    
    def optimize(self, reference_path, state, obstacles):
        # 创建新的 TEB 规划器实例
        planner = TEBPlanner(self.config, bounds)
        planner.set_global_path(reference_path)
        
        # 执行规划
        result = planner.plan(state, obstacles)
        
        # 保存规划器实例（用于访问 TEB 节点）
        self.last_planner = planner
        
        return result
    
    def update_weights(self, obstacle_weight, kinematic_weight, omega_weight):
        """动态更新 TEB 权重"""
        self.config.w_obstacle = obstacle_weight
        self.config.w_kinematics = kinematic_weight
        self.config.w_omega = omega_weight
```

#### 3.3.5 优缺点

**优点**:

### 4.4 核心差异总结

#### DWA vs TEB

**DWA 更擅长**:
- 快速响应动态变化
- 紧急避障
- 高频率控制
- 简单场景

**TEB 更擅长**:
- 复杂约束优化
- 轨迹平滑
- 时间最优
- 静态精细规划

#### Adaptive TEB-DWA 的独特优势

1. **智能融合**: 不是简单的"TEB+DWA"，而是通过反馈循环让两者协同
2. **自适应性**: 根据环境密度、路径曲率动态调整规划策略
3. **鲁棒性**: 多层次失效保护 (TEB失败→DWA兜底→最佳候选)
4. **参数自调整**: 减少对人工精细调参的依赖

## 5. 果园场景优势

### 5.1 果园环境特点

果园自主导航面临的独特挑战：

1. **结构化但复杂**: 
   - 树木呈规则排列，但分支、树干形状不规则
   - 行间宽度有限 (通常 2-4 米)
   - 需要精确跟踪行中心线

2. **静态为主，局部动态**:
   - 主要障碍物是静态的 (树木、支架)
   - 偶尔出现动态障碍 (工人、其他机器)
   - 果实、树叶可能垂落到行间

3. **对平滑性要求高**:
   - 喷洒、采摘作业需要稳定运动
   - 急加速/急转向影响作业质量
   - 震动影响传感器精度

4. **安全边距小**:
   - 需要接近树木作业
   - 容错空间小
   - 碰撞代价高 (损坏设备或果树)

### 6.3 算法选择决策树

```
开始
  │
  ├─ 是否需要极致实时性 (>50Hz)?
  │   └─ 是 → DWA
  │
  ├─ 是否纯静态环境且对平滑度要求极高?
  │   └─ 是 → TEB
  │
  ├─ 是否果园/结构化农业环境?
  │   └─ 是 → Adaptive TEB-DWA ⭐
  │
  ├─ 是否频繁出现动态障碍?
  │   └─ 是 → Adaptive TEB-DWA 或 DWA
  │
  ├─ 是否对调参时间有严格限制?
  │   └─ 是 → DWA
  │
  └─ 其他复杂场景 → Adaptive TEB-DWA
```

### 6.4 实际部署建议

#### 场景1: 标准果园作业 ⭐ 推荐

**推荐**: Adaptive TEB-DWA

**理由**:
- ✅ 平滑轨迹保证作业质量
- ✅ 自适应窗口适应不同果园布局
- ✅ 动态避障处理工人/设备
- ✅ 多层次安全保证
- ⚠️ 需要初期调参 (~2-3天)

**配置策略**:
```yaml
local_planner:
  type: adaptive_teb_dwa
  
# 使用果园优化配置 (见 5.4)
adaptive_teb_dwa:
  planner:
    max_feedback_iterations: 3
  adaptive_window:
    base_lookahead: 6.0
  # ... (完整配置见 config.yaml)
```

#### 场景2: 简单果园 + 低成本硬件

**推荐**: DWA

**理由**:
- ✅ 计算量小，低端处理器可运行
- ✅ 调参相对简单
- ✅ 稳定可靠
- ⚠️ 轨迹平滑度一般

**配置策略**:
```yaml
local_planner:
  type: dwa

dwa:
  dt: 0.1              # 降低频率以适应低端硬件
  speed_samples: 5     # 减少采样以降低计算量
  steer_samples: 11
```

#### 场景3: 研究/验证阶段

**推荐**: 三种都测试

**测试流程**:
1. 先用 **DWA** 快速验证基本功能
2. 用 **TEB** 测试最优轨迹质量
3. 用 **Adaptive** 评估综合性能
4. 对比数据，选择最优方案

### 7.3 常见问题与解决

#### 问题1: 规划器频繁失败

**症状**: 车辆停止，日志显示 "no collision-free trajectory"

**可能原因**:
- 障碍物安全边距太大
- 采样分辨率太低
- 全局路径质量差

**解决方案**:
```yaml
# 方案1: 减小安全边距
environment:
  obstacle_safety_margin: 0.3    # 从 0.4 减小到 0.3

# 方案2: 增加采样
dwa:
  speed_samples: 9               # 从 7 增加到 9
  steer_samples: 25              # 从 19 增加到 25

# 方案3: 优化全局路径
path_optimization:
  enabled: true                  # 确保启用
  max_iterations: 150            # 增加优化迭代
```

#### 问题2: 轨迹不平滑，抖动严重

**症状**: 车辆频繁左右摆动，速度波动大

**解决方案**:
```yaml
# DWA 用户:
dwa:
  steering_change_cost_weight: 1.5  # 增大
  speed_cost_weight: 0.8            # 增大
  predict_time: 3.0                 # 增大前瞻

# Adaptive 用户:
adaptive_teb_dwa:
  teb_overrides:
    w_acceleration: 10.0            # 增大
    w_omega: 15.0                   # 增大
```

#### 问题3: 计算时间过长，实时性不足

**症状**: 帧率低，控制延迟明显

**解决方案**:
```yaml
# 方案1: 降低采样数
dwa:
  speed_samples: 5
  steer_samples: 11

# 方案2: 减少 TEB 节点
teb:
  num_samples: 15
  max_iterations: 30

# 方案3: 使用缓存 (Adaptive)
adaptive_teb_dwa:
  replan_interval_steps: 3        # 从 2 增加到 3
```

#### 问题4: 动态避障响应慢

**症状**: 对移动障碍物反应迟缓，容易碰撞

**解决方案**:
```yaml
# 方案1: 切换到 DWA 或 Adaptive
local_planner:
  type: adaptive_teb_dwa          # 不要用纯 TEB

# 方案2: 缩短预测时间
dwa:
  predict_time: 1.5               # 从 2.8 减小

# 方案3: 提高更新频率
dwa:
  dt: 0.05                        # 从 0.1 减小

# 方案4: 强制每周期规划 (Adaptive)
adaptive_teb_dwa:
  replan_interval_steps: 1        # 动态环境下自动为 1
```

## 8. 总结

### 8.1 核心结论

**针对果园自主导航场景，Adaptive TEB-DWA 是最优选择**，因为：

1. **平滑性**: TEB 优化保证轨迹平滑，满足作业质量要求
2. **实时性**: DWA 评估快速，满足 10-20Hz 控制频率
3. **鲁棒性**: 多层次失效保护，适应复杂果园环境
4. **自适应**: 动态调整规划策略，无需频繁手动调参
5. **安全性**: 双重验证机制 (TEB 优化 + DWA 评估)

### 8.2 三种算法的定位

| 算法 | 最佳应用场景 | 核心优势 | 主要限制 |
|------|------------|---------|---------|
| **DWA** | 开阔动态环境<br>低成本硬件 | 极快 (5-15ms)<br>简单可靠 | 轨迹不平滑<br>局部最优 |
| **TEB** | 静态精细规划<br>研究验证 | 轨迹最平滑<br>时间最优 | 计算慢 (50-200ms)<br>收敛不保证 |
| **Adaptive** | **果园作业**<br>复杂混合场景 | 综合性能最佳<br>自适应调整 | 参数较多<br>需要调优 |

### 8.3 使用建议

**新用户**:
1. 先用 DWA 快速验证系统可行性
2. 切换到 Adaptive TEB-DWA 观察质量提升
3. 根据实际需求精细调参
